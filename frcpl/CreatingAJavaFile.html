Rules When Creating A Files

Naming Files and Classes

The following conditions must be met when naming a Java file and declaring a class. Otherwise the Java Virtual Machine (JVM) will complain.

The name of your class file must be one word altogether as there may be no spaces. It is encouraged you use "camel casing". For example: "ThisIsAnExample".

Please note that using underscores or hyphens in a manner similar to "This_is_an_example" or "This-is-an-example" is allowed, but discouraged.

When saving your file, save it with the extension name “.java”. This indicates Windows to treat this file differently from your average text file.

Please note this is not necessary for Unix-like systems such as macOS or Linux, because these systems are far superior to Windows and do not require the user to spell out the file type for them.

Example of how to declare a Java a main class.

public class nameOfYourFile {
 public static void main(String[] args) {
  //Your code here.
 }
}
